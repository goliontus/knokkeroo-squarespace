IN DEVELOPMENT — NOT TESTABLE YET

v4.x — notifications, Quick Access widget & Wear OS companion

Code lives on the main branch only. Database migrations and edge functions are now live on the backend (confirmed 20 Jul 2026) — the remaining gap is entirely client-side: no Play build has been produced from this code yet.

What's built: calm push notifications (≤1 unsolicited/day, quiet hours respected), a free-for-everyone home-screen/lock-screen Knok widget, favourite-contact status dot + own-interval greyscale nudge, a Wear OS watch companion (Kotlin) that has been verified to compile and assemble alongside the phone app — but has never been run on a watch or emulator, and its Play Store distribution path (a separate signed AAB via Play Console's Wear OS release flow) isn't configured yet — and client version tracking with a dismissible "please update" nag (soft only, never blocks the app).

Device press-time recording + offline-safe check-ins (new, 21 Jul 2026): every check-in surface now records the actual moment you pressed the button, not the moment the server happened to receive it — important because a phone can sit disconnected for a while before it reconnects and the knok finally goes through. If a device is offline when you press Knok, the check-in is saved on-device immediately and sent automatically the next time it reconnects, with a single-pending-slot design that prevents bulk-resending or spamming the server when connectivity comes back after being flaky (built with South Africa/Nigeria's patchier mobile reception specifically in mind). While a check-in is waiting to send, your own status shows a distinct blue colour plus an asterisk (not colour alone, for accessibility) so you can tell "saved, not yet confirmed" apart from a normal confirmed knok. The Terms & Conditions were updated (v2.4) to cover this, including what happens if a device's clock is wrong: if a check-in arrives with a future timestamp, the server records the actual receipt time instead.

YOUR NEXT STEP: decide when you want this tested, then tell Claude Code: "Apply the v4 database migrations, deploy the updated edge functions, and build a v4 Android test release." For the Wear OS watch app specifically, you can test it sooner and separately by telling Claude Code: "Build a debug Wear OS APK I can sideload to a watch or emulator."

⌚ Garmin

Connect IQ Store · a companion watch app, separate from the phone apps above

SUBMITTED FOR STORE REVIEW — NOT YET DOWNLOADABLE

KNOK watch companion

One button on the watch: press KNOK, done. No contacts, no status, no admin — same accessibility-first idea as the Apple TV and Wear OS companions.

Pairing works like Apple TV's: the watch shows a 6-digit code, you type it into the phone app once (Profile → Pair a Garmin), and the watch keeps a long-lived credential after that — no phone number or PIN needed on the watch itself. The backend for this (pairing + recording the knok) is built and has been tested live, end to end, against the real server — request a code, confirm it, poll for the credential, press KNOK, confirm a retry doesn't double-record, confirm a wrong credential is rejected, and confirm a lost connection saves the press time and retries automatically next time the watch reopens (same offline-safe pattern as the phone/widget/Watch/TV — see the Android v4.x section above). All of that passed.

Real build achieved, 21 Jul 2026: the Connect IQ SDK is now installed and the watch app (written in Garmin's own language, Monkey C) compiles cleanly across 135 watch models (220/220 individual device targets built successfully) — from your Fenix 7X down to entry-level models, so almost any Garmin smartwatch owner will be covered, not just one device. That produced a real, signed store package. It has been submitted to Garmin's developer dashboard for review, but hit a submission-side snag: Garmin's upload form rejected the app icon with a misleading "invalid format" error even though the icon matched Garmin's own published spec (most likely an undocumented alpha-transparency rule) — the icon was re-exported and re-submitted, and a bug report was filed with Garmin (forum thread, status "Under Review" as of 21 Jul 2026).

YOUR NEXT STEP: nothing required right now — this is sitting with Garmin (both the store review and the bug report). Worth checking back on the forum thread periodically. If it stalls, tell Claude Code: "draft an email to ConnectIQAppAdmin@garmin.com about the stuck submission" and I'll write it for you to send.
